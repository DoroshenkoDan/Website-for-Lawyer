# Сторінки практик — дизайн

- **Дата:** 2026-07-03
- **Статус:** узгоджено, готово до планування реалізації
- **Тема:** наповнення сторінок окремих практик + особлива секція порад для e-декларування

## Контекст

Усі 8 сторінок практик зараз рендеряться через плейсхолдер
[`views/PracticeArticle/PracticeArticle.tsx`](../../../views/PracticeArticle/PracticeArticle.tsx),
який виводить лише заглушку. Реального контенту немає — у messages є тільки
`practices.items.<id>.title` та `.short`.

Наявні будівельні блоки:
- [`PageBanner`](../../../components/layout/PageBanner/PageBanner.tsx) — темна hero-смуга з breadcrumb + H1 + лід + печатка RG.
- [`ContactCta`](../../../views/PracticesPage/Sections/ContactCta/ContactCta.tsx) — CTA сторінки-**списку** практик («Не впевнені, який напрям ваш?»).
- Маршрути: `app/[locale]/practices/<id>/page.tsx` для кожної практики; `electronic-declarations` додатково має підмаршрут `[category]` (наразі TODO).

Сайт тримовний: `uk` / `en` / `pl` (next-intl, `messages/<locale>.json`).

## Мета

## Структура сторінки

Спільна композиція в оновленому `PracticeArticle`:

```
<PracticeHero id={id} />          — темна смуга-заголовок + світлий блок «фото | текст»
{id === "electronic-declarations" && <PracticeTips />}
<PracticeCta />                   — новий спільний CTA (темна смуга)
```

Усі практики, окрім `electronic-declarations`, дають однакову композицію без `PracticeTips`.

### 1. `PracticeHero` (нова секція, замінює `PageBanner` на цих сторінках)

Двошарова структура «темний заголовок / світле тіло»:

```
▓▓▓ темна hero-смуга ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
▓ Breadcrumb: Головна / Практики / …    ▓
▓ НАЗВА ПРАКТИКИ (H1, світлий текст)     ▓
▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
┌──────────────┬──────────────────────┐  (світле тло сторінки)
│              │  Основний текст       │
│    ФОТО      │  практики: абзаци,    │
│              │  підзаголовки,        │
│              │  виділення…           │
└──────────────┴──────────────────────┘
```

- Темна смуга — з hero-родини (memory: banners = lifted dark hero), у стилі наявного
  `PageBanner`: breadcrumb + H1 + акцентне підкреслення. Опис/лід **не** тут — він
  переїжджає в текстову колонку нижче.
- Світле тіло — дві колонки на `lg+`: фото ліворуч (`~40%`), повний текст практики
  праворуч (`~60%`), темний текст на світлому тлі.
- Заголовок на всю ширину зверху навмисно (щоб короткий H1 поруч із високим фото не
  лишав порожнечі).
- **Мобільний/планшет:** заголовок → фото → текст, стеком.
- Фото — per-practice; поки нема реального, плейсхолдер того ж співвідношення сторін.

### 2. `PracticeTips` (тільки `electronic-declarations`)

Лонгрід зі змістом. Секція між текстом і CTA.

**Десктоп (`lg+`)** — дві колонки:
- **Ліва (~28%), липка** (`sticky top-24`): заголовок «Зміст» + нумерований список 9
  порад. Активний пункт підсвічується акцентом під час скролу (**scroll-spy** через
  `IntersectionObserver`); клік → плавний перехід (`scroll-margin-top` під липку шапку).
- **Права (~72%):** пронумеровані секції **01–09**. Кожна = номерний бейдж (акцент) +
  заголовок поради (H3) + насичене тіло, відрендерене `RichContent`.

**Мобільний/планшет:** зміст згортається у клікабельне випадне меню «Перейти до
поради ▾» зверху секції; секції йдуть стеком.

**Технічно / SEO:**
- Усі 9 порад рендеряться на **сервері** — весь текст у DOM, індексується.
- Клієнтський лише зміст зі scroll-spy (`"use client"`).
- Кожна порада має стабільний `id`-якір для прямого посилання (напр. `#who-must-file`).

### 3. `PracticeCta` (новий спільний CTA)

Один CTA на всі 8 сторінок; дизайн **відмінний** від `ContactCta`:
- **Темна hero-смуга** на всю ширину, вміст по центру: заголовок + короткий підтекст +
  головна кнопка «Записатися на консультацію» (+ email-лінк).
- Обґрунтування: темна смуга-заголовок зверху + темна CTA-смуга знизу «обрамляють»
  світлий контент (візуальний букенд).
- Текст один на всі практики, у messages: `practices.article.cta` (`title`, `text`,
  `button`).

## Модель контенту

Розподіл: **короткі рядки → messages**, **насичений довгий контент → контент-модулі**.

- Лишаються в messages (уже вживані картками/меню/SEO): `practices.items.<id>.title`,
  `.short`. Додається `practices.article.cta.{title,text,button}`.
- Насичений контент (тіло практики + поради) — у типізованих модулях під `content/`,
  бо підзаголовки/списки/виділення у пласкому JSON незручні.

### Типи

```ts
type Locale = "uk" | "en" | "pl"
type Localized<T> = Record<Locale, T>

// Інлайн-виділення пишемо маркером **слово** усередині тексту;
// RichContent рендерить його акцентним кольором (палітра), не жирним.
type Block =
  | { type: "paragraph"; text: string }
  | { type: "subheading"; text: string }
  | { type: "list"; ordered?: boolean; items: string[] }
  | { type: "callout"; variant: "example" | "note"; text: string }

interface PracticeContent {
  image: string                 // напр. /images/practices/<id>.jpg (плейсхолдер поки що)
  imageAlt?: Localized<string>
  body: Localized<Block[]>
}

interface Tip {
  id: string                    // slug для якоря, напр. "who-must-file"
  title: Localized<string>
  body: Localized<Block[]>
}
```

### Файли

- `content/practices/<id>.ts` — `PracticeContent` для кожної практики.
- `content/practices/electronic-declarations.tips.ts` — `Tip[]` (9 порад).
- Лоадери (напр. `getPracticeContent(id)`, `getDeclarationTips()`) віддають контент
  за id; за потреби — фолбек локалі на `uk`.

### Мови

Замовник дає **український** текст → наповнюємо `uk` і перекладаємо в `en` / `pl`.
Якщо якоїсь мови бракує — фолбек на `uk`.

## Спільний рендерер — `RichContent`

Один компонент, що мапить `Block[]` на розмітку; вживають і тіло практики, і поради.

- `paragraph` → `<p>` з інлайн-виділенням (`**…**` → акцентний колір із палітри, не жирний).
- `subheading` → `<h_>` відповідного рівня.
- `list` → `<ul>` / `<ol>`.
- `callout` (`example` | `note`) → блок у рамці з акцентом.

Кольори — лише з палітри (memory: use-palette-colors-only).
